The response you will receive for getting the profile for all users or for one specific user has the same structure except that the array with profile objects in the response for one specific user will only contain one object for that specific user.

Response structure
This is an overview of all fields that can be inside an user profile object. When a field is empty or not public it will not be visible in the response.

Field Type Description
count Number The number of items.
items Object[] A list of user profiles (Array of Objects)
username String Email address of the user
name String First name of the user
surname String Last name of the user
jobTitle String Job title of the user
roleCategory String Role category of the user
phone String Phone number of the user
groups Object[] List of the groups of the user (Array of Objects)
name String Name of the group (related to the type)
code String Code of the group (related to the type)
type String Type of the group, can be "Research Line", "Facility", "Directorate", "Institute".
center Object Object with center information
name String Name of the center
code String CDR code of the center
offices String[] List of office names (Array of Strings)
socials Object Object with social media links of the user
linkedin String URL of the linkedin page of the user
twitter String URL of the twitter page of the user
facebook String URL of the facebook page of the user
instagram String URL of the instagram page of the user
researchgate String URL of the research gate page of the user
googleScholar String URL of the Google Scholar page of the user
github String URL of the github page of the user
bitbucket String URL of the bitbucket page of the user
youtube String URL of the youtube page of the user
flickr String URL of the flickr page of the user
image String URL of the user profile image
titles String[] List of titles (Array of Strings). The order has been set by the user.
description String Description of the user
role String Name of the role of the user
website String URL of the website of the user
address String Address location of the user
interests String[] List of interests (Array of Strings). The order has been set by the user.
experiencesExternal Object[] List of experiences outside of IIT (Array of Objects). This array is been sorted with from and to date newest on top.
jobTitle String Job title of the experience
company String Company name of the experience
from Timestamp Start date of the experience (ISO 8601), user input format is 'mm/yyyy'
to Timestamp End date of the experience (ISO 8601), user input format is 'mm/yyyy', will be deleted if it is the current experience.
location String Location of the experience
country String Country of the experience
jobDescription String Description of the experience
education Object[] List of education items (Array of Objects). The order has been set by the user.
institute String Institute name of the education item
title String Title of the education item
from Timestamp Start date of education item (ISO 8601), user input format is 'yyyy'
to Timestamp End date of education item (ISO 8601), user input format is 'yyyy', will be deleted if it is the current education.
location String Location of education item
country String Country of education
certificates Object[] List of certificates (Array of Objects). The order has been set by the user.
title String Title of the certificate
description String Description of the certificate
date Timestamp Date of the certificate (ISO 8601), user input format is 'dd/mm/yyyy', will be deleted if no date is been provided.
favorite Boolean The certificate is favorite one of the user.
skillCategories Object[] List of skill categories of the user (Array of Objects). The order has been set by the user.
skills Object[] List of skills of the category
value String Name of the skill
favorite Boolean The skill is a favorite one of the user.
categoryName String Name of the skill category
hidden Boolean The profile is hidden (true) or visible (false).
publicWebsite Object Personal settings of the user regarding the public website
favoritePublications Boolean Value to show a section with favorite publications on the public website
allPublications Boolean Value to show a section with publications on the public website
disseminationTalks Boolean Value to show a section with dissemination talks on the public website
scientificTalks Boolean Value to show a section with scientific talks on the public website
oralPresentations Boolean Value to show a section with oral presentations on the public website
awardAchievement Boolean Value to show a section with achievements of type award on the public website
organizedEvent Boolean Value to show a section with organized events on the public website
editorship Boolean Value to show a section of editorship on the public website
experiences Object[] List of experiences within and outside of IIT (Array of Objects). Although for the moment it will only be the experiences outside of IIT due privacy settings. This array is been sorted with from and to date newest on top.
jobTitle String Job title of the experience
company String Company name of the experience
from Timestamp Start date of the experience (ISO 8601), user input format is 'mm/yyyy'
to Timestamp End date of the experience (ISO 8601), user input format is 'mm/yyyy', will be deleted if it is the current experience.
location String Location of the experience
country String Country of the experience
jobDescription String Description of the experience
id Number The users research entity ID
active Boolean The user is a current employee (true) or a former employee (false).

Not public profile
When a profile is not public you will receive an object with just two properties: 

    {
    "id": 3244
    "hidden": true
}

Example
An example of a public profile can be like: 

    {
    "count": 1,
    "items": [
        {
            "username": "john.doe@iit.it",
            "name": "John",
            "surname": "Doe",
            "jobTitle": "Research Support Administrative Senior",
            "directorate": "Research Organization Directorate",
            "office": "Data Analysis Office",
            "publicWebsite": {
                "favoritePublications": true,
                "allPublications": true,
                "disseminationTalks": true,
                "scientificTalks": true,
                "oralPresentations": true,
                "awardAchievement": true,
                "organizedEvent": true,
                "editorship": true
            },
            "id": 2295,
            "active": false
        }
    ]
}
